When the merge conflict happened, I saw conflict markers like `<<<<<<< HEAD`, `=======`, and `>>>>>>>` in the file. I decided which side to keep by carefully comparing both versions and choosing the one that best matched my intended changes. Next time, I would pull the latest changes from the remote repository before making edits to reduce the chance of conflicts happening in the first place.
